# 第 0 章：前言与使用指南

## 本章想回答什么问题？
如果一个经训练完毕、权重锁定的深度学习大模型是一个固定函数 $F(x)$，那么当我们给它输入全新的测试数据（不仅是单个样本，而是一个时序的、连续带有情景的输入序列流）时，它有可能还在“学习”吗？ 这一部分是否可以被架构层面上解构？

## 为什么这条线很重要
自 ChatGPT 以来，人们惊叹于模型能够表现出极强的 In-Context Learning (ICL) 能力。表面上看，这只是巨量参数对文本分布的一种完美背诵和顺推。然而，最近几年的研究开始逐渐揭示一层更深层次的本质：前向推理过程，甚至可以是某种退化或者变形的“优化/学习过程”本身。

如果模型在推理阶段天然具备了自适应学习的能力，这就意味着我们在训练阶段不仅要教会它世界知识，更要教会它“如何在没有额外梯度的帮助下，像真正的学习器那样提取新知识”。

## 本书不是什么
1. **不是另一本 Transformer 调参指南**：市面上谈论注意力头和并行计算的书已经汗牛充栋，我们不谈这些实现细节。
2. **不是纯粹晦涩的统计学习理论（SLT）教材**：我们不讲 PAC 界，不写长篇大论的收敛性下界定理。数学被控制在了理解核心机制的最低必要限度上（即“知道梯度、外积和优化”就足够）。

## 如何阅读本书
这是一本从最直接的工程问题（模型遇到了分布变化怎么办？）出发，顺流而下讲述我们是如何打破训练与推理的二元边界，直到揭示一切模型结构都是某种层级参数优化的书。
本书每章都有核心推导公式和一个贯穿使用的简单模型（Toy Problem，例如在线线性回归或键值索引），请务必在脑海中运行那个玩具模型。

0.2 请简单回顾深度学习中最基本的梯度下降公式并写出。
