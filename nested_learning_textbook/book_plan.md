# 教材总设计说明 (Book Plan)

## 1. 教材定位
这是一本聚焦于“**推断期学习 (Inference-Time Learning / Test-Time Learning)**”主题的针对性教材。它不是另一本关于 Transformer 基本原理的入门大全，也不是系统完整的统计学习理论（Statistical Learning Theory）教材。本教材旨在探究：**深度模型在推断/测试阶段，如何动态地更新其状态进而完成“学习”过程？** 本书以 Nested Learning (NL) 作为终局视角进行收束，提供自底向上的推断期学习演化脉络。

## 2. 目标读者
计算机科学（CS）或相关专业的高年级本科生。
**前置条件**：有使用 PyTorch/TensorFlow 进行基本模型搭建（“调包”）的经验，了解梯度下降和基础的前馈神经网络，但不要求学过复杂的 VC 维、PAC 学习等系统学习理论。

## 3. 学习目标与预期能力
读者在完成本书的阅读与实践后，应具备以下能力：
- 能够打破“训练和推理是截然分开的两个阶段”的思维定势，理解模型状态和参数的界限是模糊的。
- 掌握在测试时对模型进行更新（动态评估/TTT）的直觉和动机。
- 能够从不同的视角理解 Attention 机制（如线性注意力与 Fast Weights 的等价性，ICL 与梯度下降的同构性）。
- 理解为什么需要除了短程注意力以外的“动态持久化外部记忆（例如 Titans）”，并在脑中小心区分不同时间尺度的学习过程。
- 具备使用在线优化、联想记忆等统筹框架（如 Nested Learning）评价最新 AI 架构演进的眼光。

## 4. 主线设计逻辑
**为什么主线要从 TTT (Test-Time Training) 讲到 Nested Learning?**
TTT 是对“推理时做梯度下降”最为直白和易懂的演示，从应对数据分布偏移（Distribution Shift）这一实际问题出发，对于本科生来说非常踏实。而 Nested Learning (NL) 则是目前最宏大的一套能够把元学习、联想记忆、模型即优化器收拢起来的理论框架。从具象的工程手段（TTT）起步，穿越架构演变（Fast Weights -> Attention -> TTT layers -> Titans），最终抵达高维的视角（Nested Learning），符合人类认知规律。

**为什么不把 Nested Learning 当作一开始的起点？**
Nested Learning 是一个高度抽象、带有一种“万物皆优化”视角的框架。如果第一章就抛出“多层级双层优化”或“Context Flow”，会使初学者淹没在形式化符号中不知所云。读者必须先看过“具体哪些模型表现出了这样的行为”（例如 ICL 可以被证伪或证实为 GD，Linear Attention 等价于线性回归），其后给出大一统视角才会有“原来如此”的顿悟。

**为什么要明确区分两条 TTT 线索？**
历史上 TTT 存在两个具有代表性但范式截然不同的时期：
- **TTT 2020 范式**：测试样本到来时，通过反向传播（Backprop）针对一个自监督 auxiliary task 更新主网络的实参（Weights）。这很直观，但非常慢。
- **TTT 2024/2025 范式 (TTT layers)**：把隐藏状态（Hidden State）本身当作一个微型学习器（Learner），在测试序列上基于输入进行序列级的推断阶段持续更新。这不需要外部反向传播，而是内化在了架构前向传播之中。
如果不加区分，学生会混淆“通过外层梯度更新参数”和“通过前向过程更新基于状态的隐式参数”（例如 RNN/Mamba/TTT-Layer 的状态转移）。

## 5. 章节依赖关系
1. 第 2、3 章（动态评估与双层优化）提供后续章节的基础词汇。
2. 第 4、5 章（快权重与注意力）在具体架构上打破界限。
3. 第 6 章（ICL 是不是学习）属于视角的转折，引导读者从功能角度思考。
4. 第 7、8、9 章是现代架构实践，由浅入深讲解 Learn to Learn 在 TTT Layers 和 Titans 中的体现。
5. 第 10、11 章在所有组件齐备后，进行终局视角的统一拔高（Nested Learning）。
