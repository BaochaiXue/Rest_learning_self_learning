\section{本章想回答什么问题}
\label{sec:ttr_question}
%─────────────────────────────────────────────

我们在这本书中走过了一段漫长的旅程：
\begin{itemize}
  \item 第~\ref{ch:fastweights}~章：快权重用外积 $\mS_t = \mS_{t-1} + v_t k_t^\top$ 写入联想记忆。
  \item 第~\ref{ch:attention}~章：线性注意力的递推公式与快权重严格等价。
  \item 第~\ref{ch:icl}~章：ICL 可以被理解为隐式梯度下降。
  \item 第~\ref{ch:ttt_layers}~章：TTT-Layer 把隐状态变成通过梯度下降更新的学习器。
  \item 第~\ref{ch:titans}~章：Titans 用 Surprise 门控选择性写入神经记忆。
\end{itemize}

现在的问题是：\textbf{这些模型真的互不相干吗？}

本章将展示：\textbf{它们都是在近似求解同一个优化问题}——
\textbf{Test-Time Regression（TTR，测试时回归）}。
区别仅在于使用的优化算法、截断策略和正则化方式。

理解了这个统一视角，你就不再是``学习了十种新架构''，
而是``掌握了一个能生成所有变体的方程族''。

%─────────────────────────────────────────────
\section{最小例子：在线联想回忆的统一性}
\label{sec:ttr_minimal}
%─────────────────────────────────────────────

\begin{toybox}[title=玩具问题 F：从同一个目标函数推导出不同模型]
设一个序列模型依次观测 $(k_1, v_1), (k_2, v_2), \ldots, (k_t, v_t)$，
目标是维护一个状态矩阵 $\mS_t$，
使得对任意查询 $q$，输出 $\hat{v} = \mS_t q$ 能尽可能准确地回忆出对应的值。

定义\textbf{带遗忘的岭回归}目标：
\begin{equation}
  \mathcal{L}_t(\mS) = \frac{1}{2}\sum_{i=1}^{t} \lambda^{t-i} \|\mS k_i - v_i\|_2^2
  + \frac{\gamma}{2}\|\mS\|_F^2
  \label{eq:ttr_toy}
\end{equation}

现在观察以下三种``求解策略''的结果：
\begin{enumerate}
  \item \textbf{一步 SGD，忽略预测反馈}（$\mS_{t-1} k_t \approx 0$）：
    $\mS_t = \mS_{t-1} + v_t k_t^\top$
    $\Rightarrow$ \textbf{快权重 / 线性注意力}。

  \item \textbf{一步 SGD，保留预测反馈}：
    $\mS_t = \mS_{t-1} - \eta(\mS_{t-1} k_t - v_t) k_t^\top$
    $\Rightarrow$ \textbf{TTT-Linear}。

  \item \textbf{一步 SGD + Surprise 门控}（仅当 $\|\mS_{t-1} k_t - v_t\|$ 大时更新）：
    $\Rightarrow$ \textbf{Titans 的长时记忆}。
\end{enumerate}

\textbf{结论}：
三个看似不同的``模型架构''，
其实都是对\textbf{同一个回归目标}的不同近似求解！
\end{toybox}

%─────────────────────────────────────────────
\section{直觉解释：优化的物理实体化}
\label{sec:ttr_intuition}
%─────────────────────────────────────────────

理解这个统一视角需要一个范式转换：

\begin{quote}
\textbf{不要把序列模型理解为``一种层结构''，
而要理解为``一个未被充分跑完的优化算法在硬件上的截断展开''。}
\end{quote}

具体来说：
\begin{enumerate}
  \item 我们面临的核心问题是\textbf{在线联想回忆}：
    用有限的内存（$d \times d$ 的状态矩阵）
    存储不断到来的键-值对，并在查询时高效检索。

  \item 这天然是一个\textbf{在线优化问题}：
    每来一个新数据点，无法回顾所有历史
    （内存不够），只能``就着当前状态''做修补。

  \item 不同的``修补策略''（优化算法的选择）
    导致了不同的``层结构''——
    外积写入、梯度步、门控写入……
\end{enumerate}

所以我们其实不是在``设计层''，
而是在\textbf{选择一个在线优化器的单步更新公式}，
然后把它包装成一个``网络层''。

这个观察带来了方法论上的根本转变。
在传统的深度学习研究中，序列模型的设计往往依赖于直觉和经验式的架构搜索——
研究者先提出一种新的层结构，然后在实验中验证其效果。
而在 TTR 框架下，设计过程变成了：
\textbf{先写下一个明确的回归目标函数，
然后根据计算预算和硬件约束选择一种近似求解策略}。
不同的求解策略自然导出不同的层更新公式，
其性能和效率的权衡可以在优化理论的框架内被分析和预测。
这意味着未来的序列模型创新可能不再来自于``灵机一动想出新结构''，
而是来自于``对目标函数或求解器的系统性改进''。

\section{数学形式化：Test-Time Regression}
\label{sec:ttr_math}
%─────────────────────────────────────────────

\subsection{基石定义}

\begin{definition}[Test-Time Regression (TTR)]
\label{def:ttr}
设隐态参数为 $\mS \in \R^{d_v \times d_k}$。
在时间步 $t$，输入 token $x_t$
通过慢参数（投影矩阵）转换为 $k_t = W_K x_t$ 和 $v_t = W_V x_t$。

理想状态 $\mS_t^*$ 应最小化以下加权岭回归：
\begin{equation}
  \mathcal{L}_t(\mS) = \frac{1}{2}\sum_{i=1}^{t} \alpha_i \|\mS k_i - v_i\|_2^2
  + \frac{\gamma}{2}\|\mS\|_F^2
  \label{eq:ttr_objective}
\end{equation}
其中 $\alpha_i \ge 0$ 是每个时间点的权重（可用于实现遗忘 $\alpha_i = \lambda^{t-i}$），
$\gamma$ 是 L2 正则系数。
\end{definition}

\subsection{闭式解：Recursive Least Squares (RLS)}

目标函数~\eqref{eq:ttr_objective} 是关于 $\mS$ 的二次函数，
存在闭式解：

\begin{equation}
  \mS_t^* = V_t K_t^\top \left(K_t K_t^\top + \gamma I\right)^{-1}
  \label{eq:ttr_closedform}
\end{equation}
其中 $K_t = [\sqrt{\alpha_1} k_1, \ldots, \sqrt{\alpha_t} k_t] \in \R^{d_k \times t}$，
$V_t$ 类似定义。

这个闭式解可以通过 \textbf{Sherman–Morrison 公式}递推计算：
\begin{align}
  P_t &= P_{t-1} - \frac{P_{t-1} k_t k_t^\top P_{t-1}}{1 + k_t^\top P_{t-1} k_t}
  \label{eq:rls_P} \\
  \mS_t &= \mS_{t-1} + (v_t - \mS_{t-1} k_t) k_t^\top P_t
  \label{eq:rls_S}
\end{align}
其中 $P_t \in \R^{d_k \times d_k}$ 是精度矩阵的逆
（初始化 $P_0 = \gamma^{-1} I$）。

\begin{remark}
RLS 在每步需要 $O(d_k^2)$ 的矩阵运算来维护 $P_t$。
这虽然比重新计算逆矩阵（$O(d_k^3)$）高效，
但在 $d_k$ 很大时仍然不实用。
正是这个成本驱动了我们转向更粗糙但更便宜的近似。
\end{remark}

\subsection{一阶近似：在线 SGD}

不求精确解，而是对每个新数据点 $(k_t, v_t)$ 只做一步 SGD：

\begin{equation}
  \nabla_{\mS} \ell_t = (\mS_{t-1} k_t - v_t) k_t^\top
\end{equation}

\begin{equation}
  \mS_t = \mS_{t-1} - \eta (\mS_{t-1} k_t - v_t) k_t^\top
  \label{eq:ttr_sgd}
\end{equation}

\textbf{这就是 TTT-Linear 的状态更新公式}
（第~\ref{ch:ttt_layers}~章式~\eqref{eq:ttt_linear_grad}）。

\subsection{极端截断：忽略预测反馈}

如果进一步假设 $\mS_{t-1} k_t \approx 0$
（当前状态对新键的预测约为零——
在 $\mS$ 初始化为零且键近似正交时这是合理的），
则梯度退化为：

\begin{equation}
  \nabla_{\mS} \ell_t \approx -v_t k_t^\top
\end{equation}

代入更新公式（取 $\eta = 1$）：
\begin{equation}
  \mS_t = \mS_{t-1} + v_t k_t^\top
  \label{eq:ttr_degenerate}
\end{equation}

\textbf{这就是快权重 / 线性注意力的递推公式}
（第~\ref{ch:attention}~章式~\eqref{eq:linear_attn_recurrence}）。

\begin{insight}[title=线性注意力是在线回归的``最粗暴''近似]
线性注意力不是一种``独立发明的机制''，
而是对在线岭回归目标的一步 SGD 求解中，
进一步忽略了预测反馈项 $\mS_{t-1} k_t$ 后得到的极端截断。
它之所以有时性能不足，
正是因为缺少了这个闭环纠偏项。
TTT-Layer 通过保留这个项来获得更好的记忆质量。
\end{insight}

理解这个退化关系的意义在于：
它揭示了线性注意力的\textbf{根本性局限}不是``设计得不够精巧''，
而是来自一个明确的数学近似——忽略当前状态对新键的预测。
当输入序列的键向量彼此近似正交时（例如在信息检索任务中），
这个近似是合理的，线性注意力能表现良好。
但当键向量高度相关时（例如自然语言中相邻 token 的语义重叠），
忽略预测反馈会导致严重的记忆干扰。
这解释了为什么线性注意力在长文本建模中
常常不如 TTT-Layer 或标准 Softmax 注意力。

\section{Worked Example：从 TTR 推导 TTT-Linear}
\label{sec:ttr_worked}
%─────────────────────────────────────────────

\begin{example}[从 TTR 推导 TTT-Linear 的更新规则]
\textbf{设定}：$d_k = d_v = 3$，$\gamma = 0$（无正则），$\eta = 0.1$。

时间步 $t=1$：$k_1 = (1, 0, 0)^\top$，$v_1 = (3, 1, 2)^\top$，$\mS_0 = 0_{3 \times 3}$。

\textbf{TTR-SGD 更新}：
\begin{align}
  \text{预测} &: \mS_0 k_1 = 0 \\
  \text{残差} &: v_1 - \mS_0 k_1 = (3, 1, 2)^\top \\
  \text{梯度} &: -(v_1 - \mS_0 k_1) k_1^\top = -(3, 1, 2)^\top (1, 0, 0) = \begin{pmatrix} -3 & 0 & 0 \\ -1 & 0 & 0 \\ -2 & 0 & 0 \end{pmatrix} \\
  \mS_1 &= \mS_0 - 0.1 \times \text{梯度} = \begin{pmatrix} 0.3 & 0 & 0 \\ 0.1 & 0 & 0 \\ 0.2 & 0 & 0 \end{pmatrix}
\end{align}

时间步 $t=2$：$k_2 = (0, 1, 0)^\top$，$v_2 = (0, 5, 0)^\top$。

\textbf{TTR-SGD 更新}（注意保留了预测反馈！）：
\begin{align}
  \text{预测} &: \mS_1 k_2 = (0, 0, 0)^\top \quad (\text{因为 } k_2 \text{ 的第 1 分量为 0}) \\
  \text{残差} &: v_2 - \mS_1 k_2 = (0, 5, 0)^\top \\
  \mS_2 &= \mS_1 + 0.1 \times (0, 5, 0)^\top (0, 1, 0) = \begin{pmatrix} 0.3 & 0 & 0 \\ 0.1 & 0.5 & 0 \\ 0.2 & 0 & 0 \end{pmatrix}
\end{align}

\textbf{查询}：$q = k_1 = (1, 0, 0)^\top$：
$\hat{v} = \mS_2 q = (0.3, 0.1, 0.2)^\top = 0.1 \times v_1$。
回忆不完美（因为 $\eta = 0.1 < 1$），但方向正确。

\textbf{对比线性注意力}（忽略预测反馈，$\eta = 1$）：
$\mS_2^\text{LA} = v_1 k_1^\top + v_2 k_2^\top$，
查询 $q = k_1$ 得 $\hat{v} = v_1$——精确回忆（因为 $k_1 \perp k_2$）。

\textbf{对比 TTT-Linear}（保留预测反馈，$\eta = 1$）：
$\mS_2^\text{TTT}$ 的计算与上述相同但 $\eta = 1$，也得到精确回忆。

\textbf{结论}：当键正交时，TTT-Linear 和线性注意力给出相同结果。
它们的差异在键\textbf{非正交}时体现——
TTT-Linear 的预测反馈项可以抑制干扰。
\end{example}

\section{神经科学直觉与类比边界}
\label{sec:ttr_neuro}
%─────────────────────────────────────────────

\begin{neurosciencebox}[title=神经科学直觉：Hebbian 学习与误差驱动学习的统一]

\textbf{类比对象}：
我们可以用\textbf{Hebbian 学习}和\textbf{误差驱动学习（Error-Driven Learning）}
这两种神经科学范式来帮助理解 TTR 框架中的两端特例。

在神经科学中，突触可塑性有两个经典模型：
\begin{itemize}
  \item \textbf{Hebb 规则}：``共同激活的神经元加强连接''——
    $\Delta w \propto v \cdot k$（外积形式）。
    这是一种\textbf{无监督的、正反馈的}学习规则。

  \item \textbf{Delta 规则 / LMS}：
    $\Delta w \propto (v - w^\top k) \cdot k$（预测误差形式）。
    这是一种\textbf{有闭环纠偏的}学习规则。
\end{itemize}

\textbf{共同之处（算法层面）}：

TTR 框架中，快权重/线性注意力的更新
$\mS_t = \mS_{t-1} + v_t k_t^\top$
在形式上等同于 Hebb 规则（外积写入）。
而 TTT-Linear 的更新
$\mS_t = \mS_{t-1} - \eta(\mS_{t-1} k_t - v_t) k_t^\top$
在形式上等同于 Delta 规则（误差驱动写入）。

TTR 框架把这两者统一到一个连续谱上：
它们是同一个目标函数在不同截断程度下的产物。

\end{neurosciencebox}

\begin{analogyboundarybox}[title=这个类比在哪里失效]

\textbf{一、学习信号的来源不同（信息流层）}

\begin{tabular}{p{5cm} p{8cm}}
\textbf{TTR 框架}
& $(k_t, v_t)$ 由全局慢参数 $W_K, W_V$ 从输入 $x_t$ 投影得到。
  学习目标是全局端到端训练的结果，
  已经包含了对``什么信息值得记住''的高层判断。 \\
\textbf{Hebb / Delta 规则}
& 学习信号是\textbf{局部的}——
  仅依赖突触前后的活动，没有全局目标函数。
  神经元不知道自己的学习是否有助于``整体任务''。 \\
\end{tabular}

\medskip\textbf{二、``闭环''的含义不同（实现层）}

TTT-Linear 的``预测反馈'' $\mS_{t-1} k_t$
是通过矩阵乘法精确计算的。
生物的误差信号（如小脑的攀缘纤维信号）
是通过专门的神经回路传递的，
具有延迟、噪声和容量限制。

\medskip\textbf{读者不应从这个类比推出}：
\begin{itemize}
  \item \textbf{[X]} ``线性注意力就是 Hebbian 学习''
  \item \textbf{[X]} ``TTT-Linear 实现了大脑的误差驱动学习''
  \item \textbf{[X]} ``TTR 框架解释了大脑如何学习''
\end{itemize}

\medskip
\textbf{类比成立的层次}：\textit{数学形式}——
外积 vs.\ 误差修正的二元性在两个系统中都存在。

\textbf{类比失效的层次}：信号来源（全局 vs.\ 局部）、
误差传递机制、学习的目的性均不同。

\end{analogyboundarybox}

\section{实验}
\label{sec:ttr_lab}
%─────────────────────────────────────────────

\begin{labbox}[title=Lab 10：从 TTR 目标函数推导并实验对比三种层]
\textbf{目标}：不参考任何具体论文的架构名称，
直接通过修改目标函数的求解策略推导出三种不同的状态更新层，
并用在线回归实验对比它们的性能。

\begin{enumerate}
  \item \textbf{数据}：生成 $T = 500$ 步的在线回归序列，$d = 16$，
    键非正交（随机高斯），每 100 步切换一次目标函数参数。

  \item \textbf{方法 A}（无反馈）：
    $\mS_t = \mS_{t-1} + v_t k_t^\top$。
    记录累积误差。

  \item \textbf{方法 B}（有反馈）：
    $\mS_t = \mS_{t-1} - \eta(\mS_{t-1} k_t - v_t) k_t^\top$，$\eta = 0.01$。
    记录累积误差。

  \item \textbf{方法 C}（有反馈 + 衰减）：
    $\mS_t = \lambda \mS_{t-1} - \eta(\mS_{t-1} k_t - v_t) k_t^\top$，
    $\lambda = 0.99$，$\eta = 0.01$。
    记录累积误差。

  \item \textbf{分析}：
    绘制``时间步 vs.\ 累积 MSE''曲线。
    在任务切换点附近，哪种方法恢复最快？
\end{enumerate}

\textbf{预期观察}：
方法 B 优于方法 A（闭环纠偏）；
方法 C 在任务切换后恢复最快（衰减帮助遗忘旧信息）。
\end{labbox}

\section{常见误解}
\label{sec:ttr_misconceptions}
%─────────────────────────────────────────────

\begin{misconceptionbox}
\textbf{误解 1}：既然所有模型都在解同一个优化问题，
那么它们的实际性能应该差不多。

\textbf{纠正}：
理论上同源，但``极其微小''的近似差异
会导致实际性能和硬件效率的巨大差异。
例如：线性注意力（无反馈）在 GPU 上吞吐量最高，
但记忆质量差；
TTT-MLP（有反馈 + 非线性）记忆质量好，
但每步需要对内层 MLP 做反向传播，吞吐量显著下降。
选择哪个``近似''取决于具体的硬件约束和任务需求。
\end{misconceptionbox}

\begin{misconceptionbox}
\textbf{误解 2}：TTR 框架完全解释了深度学习的一切。

\textbf{纠正}：
TTR 在\textbf{机制层面}提供了一种``优化算子''的统一语言。
但``统一了底层运算的数学形式''
不等于``解释了复杂行为的涌现''。
大模型的推理能力、多模态对齐、
对抗鲁棒性等高层现象，
至今没有被任何单一框架充分解释。
TTR 是理解机制的有力工具，不是万能理论。
\end{misconceptionbox}

\section{练习题}
\label{sec:ttr_exercises}
%─────────────────────────────────────────────

\begin{exercise}
\textbf{推导 RetNet}：
从 TTR 目标函数~\eqref{eq:ttr_objective} 出发，
设 $\alpha_i = \lambda^{t-i}$（指数衰减权重），
$\gamma = 0$，优化策略为``一步 SGD，忽略预测反馈''。
推导出 RetNet 的递推更新公式
$\mS_t = \lambda \mS_{t-1} + v_t k_t^\top$。
说明衰减因子 $\lambda$ 在目标函数中的含义。
\end{exercise}

\begin{exercise}
\textbf{RLS 的递推推导}：
使用 Sherman--Morrison 恒等式
$(A + uv^\top)^{-1} = A^{-1} - \frac{A^{-1} u v^\top A^{-1}}{1 + v^\top A^{-1} u}$，
从 $\mS_{t-1}^* = V_{t-1} K_{t-1}^\top (K_{t-1} K_{t-1}^\top + \gamma I)^{-1}$
推导出 $\mS_t^*$ 的递推更新公式~\eqref{eq:rls_P}--\eqref{eq:rls_S}。
说明为什么 RLS 虽然是``精确解''，
但在实践中不如一步 SGD 常用（提示：$P_t$ 的维护代价）。
\end{exercise}

\begin{exercise}
\textbf{Surprise 门控的 TTR 解读}：
将 Titans 中的 Surprise 值 $s_t = \|\mS_{t-1} k_t - v_t\|_2^2$
对应到 TTR 目标函数~\eqref{eq:ttr_objective} 中的哪一项？
解释为什么高 Surprise 表示``当前状态对新数据的预测很差''，
因此应该加大更新步长。
\end{exercise}

\section{本章小结}
\label{sec:ttr_summary}
%─────────────────────────────────────────────

\begin{itemize}
  \item \textbf{TTR 统一视角}：
    快权重、线性注意力、TTT-Linear、TTT-MLP、Titans 的长时记忆
    都可以被理解为对同一个在线岭回归目标的不同近似求解。
    区别仅在于优化算法、截断策略和附加门控。

  \item \textbf{核心方程}：
    TTR 目标函数~\eqref{eq:ttr_objective}
    $\to$ 一步 SGD 得到通用更新 $\mS_t = \mS_{t-1} - \eta(\mS_{t-1} k_t - v_t) k_t^\top$
    $\to$ 忽略反馈退化为外积写入 $\mS_t = \mS_{t-1} + v_t k_t^\top$。

  \item \textbf{方法论转变}：
    序列模型设计从``搭积木试错''转向
    ``从目标函数推导新变体''。
    通过修改正则化项、求解策略和门控参数，
    可以系统地推导出新的模型层。

  \item \textbf{闭式解 vs.\ 近似}：
    RLS 给出精确闭式解但计算代价高；
    一步 SGD 便宜但不精确；
    实际系统在两端之间权衡。
\end{itemize}

\paragraph{读完本章，你应该能够：}
\begin{itemize}
  \item 写出 TTR 目标函数，并从中推导出快权重、TTT-Linear 的更新公式
  \item 解释线性注意力为什么是``最粗暴的近似''
        ——因为它忽略了预测反馈项
  \item 对一个新提出的序列模型层，判断它在 TTR 框架中对应的优化策略
  \item 用 Sherman--Morrison 公式推导 RLS 的递推更新
\end{itemize}
