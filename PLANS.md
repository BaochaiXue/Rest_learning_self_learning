# PLANS.md

Use this template for any non-trivial task. Copy it into the active work file, usually `research/research_plan.md`.

## Objective

- What outcome should exist when this task is complete?

## Scope / Non-Goals

- What is explicitly in scope?
- What is explicitly out of scope?

## Inputs / Dependencies

- Which files, papers, datasets, tools, or prior decisions does this depend on?
- Which unknowns could block progress?

## Work Breakdown

1. Preparation:
2. Evidence gathering:
3. Implementation or execution:
4. Verification:
5. Logging and synthesis:

## Verification Plan

- What checks, tests, or comparisons will confirm success?
- What evidence must be written back to the repo?

## Failure / Rollback Plan

- What is the smallest safe rollback?
- What should happen if evidence is weak, results are inconclusive, or implementation fails?

## Files To Touch

- List only files that are expected to change.

## Completion Checklist

- [ ] Plan updated before substantial work
- [ ] Evidence gathered from the right sources
- [ ] Implementation or execution completed
- [ ] Verification run or blocker recorded
- [ ] `research/findings.md` updated
- [ ] Decisions and open questions updated if needed
- [ ] Final synthesis stays within verified evidence
