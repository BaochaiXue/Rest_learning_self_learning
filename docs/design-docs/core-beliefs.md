# Core Beliefs

These are the operating beliefs behind the root research OS.

1. Map before detail. Put the entrypoint and file map in front of long prose.
2. Narrow beats universal. Use small, role-specific agents instead of one large agent that does everything.
3. Evidence beats memory. Important conclusions must be backed by a file path, source, or artifact.
4. Plans are artifacts. A real task gets a plan file, not just a chat outline.
5. Handoffs are first-class. Long work needs resumable checkpoints.
6. Claims need provenance. Mark `verified` or `uncertain` and trace the support.
7. Reversibility matters. Prefer changes that can be rolled back cleanly.
8. Root docs orchestrate; child docs govern their own subtree.
9. Generated outputs should be labeled as generated and kept separate from policy.
10. Reliability is a design choice. Build for repeated long sessions, not one-off success.
