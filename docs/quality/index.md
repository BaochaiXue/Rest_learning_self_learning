# docs/quality/index.md

This directory holds root-level quality gates.
Use it when a task needs a policy for claim discipline, verification quality, or release readiness.

## Files

- `qa_rubric.md`: what “good enough” means for a root docs or research artifact
- `claim_honesty_policy.md`: how to write claims without overstating evidence

## Rule

- If a statement will be reused as a conclusion, it must pass through this directory's rules first.
