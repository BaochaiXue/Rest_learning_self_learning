# Claim Honesty Policy

Write claims the way a careful reviewer would want to read them.

## Rules

- A claim must be traceable to a source, experiment artifact, or file path.
- A claim that is not yet supported must be labeled `uncertain`.
- A conclusion without evidence belongs in `open_questions` or `decision_log`, not in the final claim.
- Do not invent citations, baselines, benchmarks, settings, or results.
- Do not collapse interpretation into fact; say when something is an inference.
- If a statement depends on a child-project fact, cite the child docs or child artifacts directly.

## Preferred Writing Pattern

- State the evidence.
- State the inference.
- State the remaining uncertainty.
