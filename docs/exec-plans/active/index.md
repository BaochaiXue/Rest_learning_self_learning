# docs/exec-plans/active/index.md

Active execution plans live here.
Create one Markdown file per active workstream, and keep it updated while the work is in flight.

## Recommended File Shape

- Objective
- Scope and non-goals
- Inputs and dependencies
- Work breakdown
- Verification plan
- Failure and rollback plan
- Files to touch
- Completion checklist

## Update Rule

- Refresh the plan before non-trivial changes.
- Add the next checkpoint whenever the work hands off to another agent or session.
- Move completed work out of `active/` only after the final evidence is written back to the repo.
